## 一、游戏概述

**类型**：口袋妖怪风格的像素风RPG游戏

**核心玩法**：
- 探索8张地图（5张主线 + 3张隐藏）
- 与30种怪物 + 9个Boss战斗
- 与15个NPC互动，获取隐藏信息
- 收集道具、装备、材料
- 解锁隐藏关卡和彩蛋

**技术栈**：
- 前端：HTML5 Canvas + JavaScript（单文件）
- 渲染：64x64像素瓦片，60fps
- 存储：LocalStorage
- 音效：Web Audio API

**操作方式**：
- ↑↓←→ 移动
- Shift 加速跑
- 空格 互动/对话/确认
- ESC 菜单
- I 背包
- M 地图
- S 快速存档

```
新手村 ←→ 翡翠草原 ←→ 幽暗森林 ←→ 烈焰火山 ←→ 魔王城堡

- 遇敌率：8%（每步）

### 6.1 战斗流程

```
玩家回合 → 选择行动 → 执行行动 → 检查怪物死亡
    ↓
怪物回合 → 怪物攻击 → 检查玩家死亡
    ↓
下一回合
```

### 6.2 伤害公式

```
基础伤害 = ATK × (1 + random(0.1)) - DEF × 0.5
属性克制 = 基础伤害 × 1.5（弱点）或 × 0.5（抗性）
暴击 = 伤害 × 1.5（10%概率）
最终伤害 = max(1, floor(伤害))
```

## 十二、技术实现

### 12.1 文件结构

```
pixel_rpg/
├── index.html              # 单文件游戏（所有代码+数据）
├── GAME_DESIGN.md          # 本设计文档
└── js/                     # 旧版多文件（已弃用，保留参考）
```

**当前实现：单HTML文件架构**
- 所有代码、数据、渲染逻辑均在 index.html 中
- 约1500行，包含：配置、技能、物品、怪物、Boss、NPC、地图、室内、输入、渲染、地图管理、玩家、怪物类、战斗、UI、存档、音频、游戏主循环

### 12.2 核心模块（单文件内）

| 模块 | 职责 | 主要类/函数 |
|------|------|---------|
| CFG | 游戏常量配置 | 瓦片类型、元素相克、遇敌率 |
| SKILLS/ITEMS/MONS/BOSS | 数据定义 | 12技能、25物品、30怪物、9Boss |
| NPCS/MAPS/INTERIORS | 世界数据 | 16NPC、8地图、5室内 |
| Input | 键盘输入 | dir(), onKey |
| Renderer | Canvas渲染 | drawTile(), drawChar(), dayNight() |
| MapMgr | 地图管理 | load(), gen(), canPass(), render() |
| Player | 玩家逻辑 | move(), enterInterior(), exitInterior() |
| Monster | 怪物类 | dmg(), getDrops() |
| Battle | 战斗系统 | start(), render(), pAtk(), useSkill() |
| UI | 界面显示 | msg(), dlgShow(), render() |
| Save | 存档系统 | save(), load() |
| Audio | 音效系统 | play(), step(), atk() |
| doorCd | 门冷却系统 | 防止进出房间循环卡死 |
| Weather | 天气系统 | 晴/雨/雪/雾，影响战斗属性 |
| TimeSys | 时间系统 | 昼夜循环，影响怪物属性 |

### 12.3 渲染流程

```
gameLoop()
  ├── update()
  │   ├── updateAnimation()
  │   ├── updatePlayer()
  │   ├── checkEncounter()
  │   ├── checkNPC()

  └── render()
      ├── renderMap()
      ├── renderNPCs()
      ├── renderPlayer()
